# AI Agent Guidance

## Overview

AI tools were used during the development of this project to accelerate coding, debugging, and documentation. The AI acted as an assistant to generate code templates, suggest improvements, and help troubleshoot issues.

The final implementation was reviewed and modified to ensure correctness, readability, and adherence to software engineering best practices.

---

## Roles of AI Agents

The AI tools were used for the following purposes:

### 1. Code Scaffolding

AI was used to generate initial project structure including:

- Flask backend setup
- SQLAlchemy model definitions
- React frontend scaffolding
- API endpoint templates

This allowed faster bootstrapping of the application.

---

### 2. Debugging Assistance

AI was used to help identify and resolve issues such as:

- Python dependency errors
- PostgreSQL connection issues
- Flask routing errors
- React API integration problems

---

### 3. Documentation Generation

AI assisted in generating:

- README documentation
- architecture explanation
- code comments
- developer guidance files

---

### 4. Best Practice Suggestions

AI recommended improvements such as:

- Using SQLAlchemy ORM instead of raw SQL
- Organizing backend files into modular components
- Using React hooks for state management
- Adding clear API endpoint structure

---

## Human Oversight

All AI generated code and suggestions were manually reviewed and tested before being included in the project.

Human review ensured:

- Code correctness
- Security considerations
- Proper architecture decisions
- Clean and maintainable code structure

---

## Limitations

AI generated suggestions can sometimes be overly generic or not optimized for specific project needs. Therefore, manual validation and testing were necessary before implementation.
